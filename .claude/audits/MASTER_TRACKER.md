## Scoring System

**Overall Score:** 0-100 based on:

- TypeScript correctness: 25 points
- SolidJS best practices: 25 points
- Accessibility: 25 points
- Code quality: 25 points

**Issue Severity:**

- **Critical:** Breaks functionality, security issues, major accessibility violations
- **High:** Performance issues, significant a11y problems, incorrect patterns
- **Medium:** Code style issues, minor a11y improvements, missing types
- **Low:** Documentation, naming conventions, minor improvements

### Key Improvements Made (2026-01-13 Batch Audit):

1. **Accessibility:** Added role attributes, ARIA labels, screen reader support across all components
2. **TypeScript:** Added return type annotations (`: JSX.Element`) to all components
3. **Documentation:** Added JSDoc comments with @default tags for all props

5. **Semantic HTML:** Fixed non-semantic elements (e.g., UploadFile remove button)
6. **Reactivity:** Fixed prop access issues (using `local.x` instead of `props.x`)

### Key Findings (2026-01-14 Hooks & Contexts Audit):

1. **Missing JSDoc:** Most hooks/contexts lack comprehensive JSDoc documentation
2. **localStorage Handling:** Several contexts don't wrap localStorage in try-catch for private browsing
3. **Type Safety:** Some contexts created without type parameter (`createContext()`)
4. **Timer Types:** Using `NodeJS.Timeout` instead of `ReturnType<typeof setTimeout>`
5. **Return Types:** Many hooks missing explicit return type annotations
6. **Strengths:** Excellent SWR pattern in useCustomResource, proper FormatJS integration, good SolidJS reactive patterns

## Commands

To audit a component:

```bash
npm run claude:audit -- ComponentName
```

To document a component:

```bash
npm run claude:docs -- ComponentName
```

To fix issues:

```bash
npm run claude:fix -- ComponentName
```
