/**
 * Hook to access the toast notification API.
 * Provides methods to show, dismiss, pause, and play toast notifications.
 * Must be used within a ToastProvider.
 *
 * @returns ToastAPI with dynamic methods based on configured toast types
 * @throws Error if used outside ToastProvider
 *
 * @example
 * ```tsx
 * const toast = useToast();
 *
 * // Show toast (method names depend on ToastProvider config)
 * toast.success('Operation completed!');
 * toast.error('Something went wrong');
 *
 * // With options
 * const id = toast.info('Loading...', { duration: 0 }); // persistent
 *
 * // Control existing toast
 * toast.dismiss(id);
 * toast.pause(id);
 * toast.play(id);
 * ```
 */
````

## Positive Findings

1. **Focused API:** Returns only `context.api`, not the entire context.
2. **Proper Validation:** Throws if used outside provider.
3. **Dynamic Methods:** The Proxy-based API allows any method name configured in the provider.

## API Surface

The returned API includes:

- `dismiss(id: string)` - Dismiss a specific toast
- `pause(id: string)` - Pause a toast's timer
- `play(id: string)` - Resume a paused toast
- `[methodName](message, options?)` - Dynamic methods based on configured toast types
