```typescript

**Problem:** No explicit return type.
**Recommendation:**

```typescript
export const useTheme = (): ThemeContextType => {
```

#### 2. Missing JSDoc Documentation

**Location:** Throughout
**Recommendation:**

````typescript
/**
 * Hook to access and control the application theme.
 * Provides system theme detection and manual theme override.
 * Must be used within a ThemeProvider.
 *
 * @returns ThemeContextType with theme accessors and setters
 * @throws Error if used outside ThemeProvider
 *
 * @example
 * ```tsx
 * const { appTheme, setAppTheme, systemTheme } = useTheme();
 *
 * // Get current theme
 * const isDark = appTheme() === 'dark' ||
 *   (appTheme() === 'system' && systemTheme() === 'dark');
 *
 * // Toggle theme
 * setAppTheme(isDark ? 'light' : 'dark');
 * ```
 */
````

### Low Severity

#### 3. ThemeContextType Not Re-exported

**Problem:** Consumers need to import from context file separately.
**Recommendation:** Re-export the type from the hook file.

## API Surface

The returned context provides:

- `systemTheme: Accessor<'dark' | 'light'>` - Detected system preference
- `appTheme: Accessor<'dark' | 'light' | 'system'>` - Current app theme setting
- `setAppTheme: (theme) => void` - Function to change theme
