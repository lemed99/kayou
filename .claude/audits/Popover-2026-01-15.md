## Executive Summary

The Popover component has critical accessibility gaps. It lacks keyboard support entirely - no way to open/close with keyboard, no focus management, and missing ARIA attributes. The click-outside logic has redundant code paths that could cause issues. The component uses good SolidJS patterns with `useFloating` and `createPresence`, but needs significant accessibility work before production use.

## Critical Priority Issues (Must Fix)

### Issue 1: No Keyboard Support

**Severity:** Critical
**Category:** Accessibility
**Line(s):** 142-156

**Current Code:**

```typescript
<div
  ref={refs.setReference}
  class="w-full"
  onClick={() =>
    !merged.hidden && !merged.onHover ? setIsPopoverVisible(true) : null
  }
  onMouseEnter={() => { ... }}
  onMouseLeave={() => { ... }}
>
```

**Problem:**
The trigger element only responds to mouse events. Keyboard users cannot:

- Focus the trigger (no `tabindex`)
- Open the popover (no Enter/Space handler)
- Close the popover (no Escape handler)

This is a WCAG 2.1 Level A failure (2.1.1 Keyboard).

**Solution:**

```typescript
<div
  ref={refs.setReference}
  class="w-full"
  tabindex={merged.onHover ? undefined : 0}
  role={merged.onHover ? undefined : "button"}
  aria-haspopup="dialog"
  aria-expanded={isPopoverVisible()}
  onClick={() =>
    !merged.hidden && !merged.onHover ? setIsPopoverVisible(!isPopoverVisible()) : null
  }
  onKeyDown={(e) => {
    if (merged.onHover || merged.hidden) return;
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      setIsPopoverVisible(!isPopoverVisible());
    }
    if (e.key === 'Escape' && isPopoverVisible()) {
      setIsPopoverVisible(false);
      props.onClose?.();
    }
  }}
  onMouseEnter={() => { ... }}
  onMouseLeave={() => { ... }}
>
```

### Issue 2: Missing ARIA Attributes

**Severity:** Critical
**Category:** Accessibility
**Line(s):** 139-159, 162-194

**Current Code:**

  {merged.children}

...

**Problem:**
Missing essential ARIA attributes for popover pattern:

- No `role` on trigger or popover
- No `aria-haspopup` on trigger
- No `aria-expanded` on trigger
- No `aria-controls` linking trigger to popover
- No `id` on popover for `aria-controls` reference

**Solution:**

```typescript
const popoverId = `popover-${createUniqueId()}`;

<div
  ref={refs.setReference}
  role={merged.onHover ? undefined : "button"}
  aria-haspopup="dialog"
  aria-expanded={isPopoverVisible()}
  aria-controls={isPopoverVisible() ? popoverId : undefined}
  ...
>

<div
  ref={refs.setFloating}
  id={popoverId}
  role="dialog"
  aria-modal="false"
  ...
>
```

### Issue 3: No Focus Management

**Severity:** Critical
**Category:** Accessibility
**Line(s):** 160-196

**Problem:**
When the popover opens:

- Focus doesn't move to the popover content
- No focus trap for complex popover content
- When popover closes, focus doesn't return to trigger

**Solution:**
Add focus management:

```typescript
let triggerRef: HTMLDivElement | undefined;

createEffect(() => {
  if (isPopoverVisible()) {
    // Focus first focusable element in popover, or the popover itself
    const floatingEl = refs.floating();
    if (floatingEl) {
      const focusable = floatingEl.querySelector<HTMLElement>(
        'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])',
      );
      (focusable || floatingEl)?.focus();
    }
  }
});

// Return focus on close
createEffect((prev) => {
  if (prev && !isPopoverVisible()) {
    triggerRef?.focus();
  }
  return isPopoverVisible();
});
```

## High Priority Issues (Fix Before v1.0)

### Issue 4: Redundant Click-Outside Logic

**Severity:** High
**Category:** Code Quality
**Line(s):** 92-129

**Current Code:**

```typescript
const handleClickOutside = (event: MouseEvent) => {
  // First check - closes for onHover mode
  if (
    merged.onHover &&
    !floatingEl?.contains(target) &&
    !referenceEl?.contains(target)
  ) {
    setIsPopoverVisible(false);
    props.onClose?.();
  }

  // Second check - guard clause
  if (!isMounted() || ...) {
    return;
  }

  // Third check - compareDocumentPosition
  if (floatingEl?.compareDocumentPosition(target) & ...) {
    return;
  }

  // Fourth action - closes again
  setIsPopoverVisible(false);
  props.onClose?.();
};
```

**Problem:**

1. In `onHover` mode, the function closes the popover at line 103-104 but doesn't return, potentially calling `onClose` twice
2. The `compareDocumentPosition` check on line 118 is confusing and may have edge cases
3. `onClose` could be called multiple times in a single click

**Solution:**

```typescript
const handleClickOutside = (event: MouseEvent) => {
  if (!isMounted()) return;

  const target = event.target as Node;
  const floatingEl = refs.floating();
  const referenceEl = refs.reference();

  // Click inside trigger or popover - do nothing
  if (floatingEl?.contains(target) || referenceEl?.contains(target)) {
    return;
  }

  // Click outside - close popover
  setIsPopoverVisible(false);
  props.onClose?.();
};
```

### Issue 6: Missing Controlled Mode

**Severity:** High
**Category:** API Design

**Problem:**
No way to control the popover externally via `isOpen` and `onOpenChange` props. This limits integration with external state management.

**Solution:**

```typescript
interface PopoverProps {
  // ... existing props
  /**
   * Controlled open state.
   */
  isOpen?: boolean;
  /**
   * Callback when open state changes.
   */
  onOpenChange?: (isOpen: boolean) => void;
}

// In component:
const [internalOpen, setInternalOpen] = createSignal(false);
const isControlled = () => props.isOpen !== undefined;
const isPopoverVisible = () => (isControlled() ? props.isOpen! : internalOpen());

const setOpen = (open: boolean) => {
  if (!isControlled()) {
    setInternalOpen(open);
  }
  props.onOpenChange?.(open);
};
```

### Issue 7: onClose Called on Unmount Even If Never Opened

**Severity:** High
**Category:** Code Quality
**Line(s):** 131-135

**Current Code:**

```typescript
onCleanup(() => {
  if (isMounted()) {
    props.onClose?.();
  }
});
```

**Problem:**
`isMounted()` checks if the popover is currently mounted, not if it was ever opened. If the popover is open when the component unmounts, this is correct. But if the popover was never opened, `onClose` shouldn't be called.

**Solution:**
Track whether the popover was ever opened:

```typescript
let wasEverOpened = false;

createEffect(() => {
  if (isPopoverVisible()) {
    wasEverOpened = true;
  }
});

onCleanup(() => {
  if (wasEverOpened && isMounted()) {
    props.onClose?.();
  }
});
```

**Estimated Effort:** 10 minutes
