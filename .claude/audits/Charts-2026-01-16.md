## Executive Summary

The Charts module provides LineChart and PieChart components built on D3.js. While the components use good SolidJS reactive patterns with `createMemo` and proper context management, they have significant accessibility gaps. Charts are notoriously difficult for accessibility, but basic ARIA roles, labels, and keyboard navigation for interactive elements are missing. Additionally, there are DOM querying anti-patterns using `document.querySelector` that could break with multiple chart instances.

## Critical Priority Issues (Must Fix)

### Issue 1: Missing SVG Accessibility Attributes

**Severity:** Critical
**Category:** Accessibility
**Component:** LineChart, PieChart
**Line(s):** LineChart.tsx:179-186, PieChart.tsx:14-19

**Current Code:**

```typescript
<svg
  width={width()}
  height={height()}
  style={{ overflow: 'visible', width: '100%', height: '100%' }}
  onMouseMove={onMouseMove}
  onMouseLeave={() => setActiveIndex(null)}
  viewBox={`0 0 ${width()} ${height()}`}
>
```

**Problem:**
SVG charts need accessibility attributes for screen readers:

- No `role="img"` or `role="graphics-document"`
- No `aria-label` describing the chart
- No `<title>` or `<desc>` elements for accessible name/description
- No `aria-hidden` to hide decorative chart elements

WCAG 1.1.1 (Non-text Content) requires text alternatives for charts.

**Solution:**

```typescript
<svg
  width={width()}
  height={height()}
  role="img"
  aria-label={props.ariaLabel ?? "Line chart"}
  aria-describedby={props.ariaDescribedBy}
  style={{ overflow: 'visible', width: '100%', height: '100%' }}
  onMouseMove={onMouseMove}
  onMouseLeave={() => setActiveIndex(null)}
  viewBox={`0 0 ${width()} ${height()}`}
>
  <title>{props.title ?? "Chart"}</title>
  <desc>{props.description}</desc>
  ...
</svg>
```

### Issue 2: DOM querySelector Anti-Pattern

**Severity:** Critical
**Category:** Code Quality / Bugs
**Component:** XAxis, YAxis
**Line(s):** XAxis.tsx:12-17, YAxis.tsx:15-21

**Current Code (XAxis):**

```typescript
onMount(() => {
  chart.setXKey(props.dataKey);
  const g: SVGGElement | null = document.querySelector('.x-axis');
  if (g) {
    const bbox = g.getBBox();
    setAxisBBox(bbox);
    chart.setXAxisBBox(bbox);
  }
});
```

**Problem:**
Using `document.querySelector('.x-axis')` will select the **first** element with that class in the entire document. If there are multiple charts on a page, this will:

1. Always return the first chart's axis, not the current chart's
2. Cause incorrect sizing calculations
3. Lead to visual bugs in multi-chart layouts

**Solution:**
Use refs instead:

```typescript
let axisRef: SVGGElement | undefined;

onMount(() => {
  chart.setXKey(props.dataKey);
  if (axisRef) {
    const bbox = axisRef.getBBox();
    setAxisBBox(bbox);
    chart.setXAxisBBox(bbox);
  }
});

return (
  <g class="x-axis" ref={axisRef}>
    ...
  </g>
);
```

**Estimated Effort:** 20 minutes per axis

- No ARIA attributes for segment data

  onMouseEnter={() => setActiveSector({...})}
  onFocus={() => setActiveSector({...})}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      // Handle selection if needed
    }
  }}
  style={{ cursor: 'pointer' }}

### Issue 5: document.getElementById Usage

**Severity:** High
**Category:** SolidJS Practices
**Component:** LineChart
**Line(s):** LineChart.tsx:145-153

**Current Code:**

```typescript
createEffect(() => {
  if (activeIndex()) {
    const box = document.getElementById(tooltipId);
    if (box) {
      const { width, height } = box.getBoundingClientRect();
      setBoxDimensions({ width, height });
    }
  }
});
```

**Problem:**
Direct DOM access via `document.getElementById` is:

1. Not the SolidJS way (should use refs)
2. Less performant
3. Potentially problematic with SSR

**Solution:**

```typescript
let tooltipRef: HTMLDivElement | undefined;

createEffect(() => {
  if (activeIndex() && tooltipRef) {
    const { width, height } = tooltipRef.getBoundingClientRect();
    setBoxDimensions({ width, height });
  }
});

// In JSX:
<div ref={tooltipRef} ...>
```

### Issue 6: LineChartTooltip Cleanup Missing

**Severity:** High
**Category:** Memory / Cleanup
**Component:** LineChartTooltip
**Line(s):** LineChartTooltip.tsx:9-12

**Current Code:**

```typescript
onMount(() => {
  const tooltipContent = () => props.content;
  chart.setCustomTooltip(tooltipContent);
});
```

**Problem:**

- Sets custom tooltip on mount but never cleans up
- If the Tooltip component unmounts, the parent LineChart still has the stale tooltip reference
- The wrapped function `() => props.content` creates unnecessary indirection

**Solution:**

```typescript
import { onCleanup, onMount } from 'solid-js';

onMount(() => {
  chart.setCustomTooltip(() => props.content);
});

onCleanup(() => {
  chart.setCustomTooltip(undefined);
});
```

**Estimated Effort:** 10 minutes

### Issue 7: Sector Math Rotation Undocumented

**Severity:** High
**Category:** Code Quality
**Component:** Sector
**Line(s):** Sector.tsx:12-13

**Current Code:**

```typescript
.startAngle(props.startAngle + Math.PI / 2)
.endAngle(props.endAngle + Math.PI / 2)
```

**Problem:**
The `+ Math.PI / 2` rotation offset is not documented. This converts from D3's default (12 o'clock start) to a 3 o'clock start, but:

1. No comment explaining why
2. Could confuse maintainers
3. Magic number anti-pattern

**Solution:**

```typescript
// D3's pie layout starts at 12 o'clock (north), but we want 3 o'clock (east)
// Rotate by 90 degrees (π/2 radians) to match design expectations
const ROTATION_OFFSET = Math.PI / 2;

const arcGen = createMemo(() =>
  arc()
    .innerRadius(props.innerRadius)
    .outerRadius(props.outerRadius)
    .startAngle(props.startAngle + ROTATION_OFFSET)
    .endAngle(props.endAngle + ROTATION_OFFSET)
    .padAngle(0.01),
);
```

**Estimated Effort:** 5 minutes

### Issue 10: PieProps dataKey Type Too Loose

**Severity:** Medium
**Category:** Type Safety
**File:** types.ts:119

**Current Code:**

```typescript
export interface PieProps {
  data: Record<string, unknown>[];
  dataKey: keyof Record<string, unknown>;  // This is just `string`
```

**Problem:**
`keyof Record<string, unknown>` evaluates to `string`, losing type safety. The intention is to constrain `dataKey` to actual keys in the data.

**Solution:**
Use generics:

```typescript
export interface PieProps<T extends Record<string, unknown> = Record<string, unknown>> {
  data: T[];
  dataKey: keyof T;
  // ...
}
```

### Issue 12: IIFE Pattern in Line Component

**Severity:** Medium
**Category:** Code Style
**File:** Line.tsx:53-69

**Current Code:**

```typescript
<Show when={chart.activeIndex() !== null}>
  {(() => {
    const activeIndex = chart.activeIndex()!;
    return (
      <For each={points().filter((p) => p.x === activeIndex.x)}>
        ...
      </For>
    );
  })()}
</Show>
```

**Problem:**
The IIFE pattern is non-idiomatic in SolidJS. The `Show` component already handles the conditional, and using an IIFE just to capture `activeIndex` is unnecessary.

**Solution:**

```typescript
<Show when={chart.activeIndex()}>
  {(activeIndex) => (
    <For each={points().filter((p) => p.x === activeIndex().x)}>
      {(p) => (
        <circle
          cx={activeIndex().x}
          cy={p.y}
          r={3}
          fill="white"
          stroke={props.stroke ?? 'currentColor'}
          stroke-width={1}
        />
      )}
    </For>
  )}
</Show>
```

### Issue 13: YAxis dy Calculation is Hacky

**Severity:** Medium
**Category:** Code Quality
**File:** YAxis.tsx:48

**Current Code:**

```typescript
dy={i() === ticks().length - 1 ? 8 : i() === 0 ? 0 : 4}
```

**Problem:**
Magic numbers for text positioning based on index. Not clear why first/last ticks need different positioning.

**Solution:**
Add comments and named constants:

```typescript
// Vertical offset for tick labels:
// - First tick (top): align bottom of text with line
// - Last tick (bottom): align top of text with line
// - Middle ticks: center vertically
const TOP_TICK_OFFSET = 0;
const MIDDLE_TICK_OFFSET = 4;
const BOTTOM_TICK_OFFSET = 8;

const getTickDy = (index: number, total: number) => {
  if (index === 0) return TOP_TICK_OFFSET;
  if (index === total - 1) return BOTTOM_TICK_OFFSET;
  return MIDDLE_TICK_OFFSET;
};

// In JSX:
dy={getTickDy(i(), ticks().length)}
```

**Estimated Effort:** 10 minutes
